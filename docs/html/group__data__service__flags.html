#define IDIGI_MSG_FIRST_DATA   0x0001

This flag is used in to indicate that this is the first message for this data transfer.

See also:
idigi_data_service_put_request callback
idigi_data_service_device_request callback First chunk of data
#define IDIGI_MSG_LAST_DATA   0x0002

This flag is to indicate that this is the last message for this transfer.

See also:
idigi_data_service_put_request callback
idigi_data_service_device_request callback Last chunk of data
#define IDIGI_MSG_DATA_NOT_PROCESSED   0x0010

This flag is used to indicate tha the message was not processed.

See also:
idigi_data_service_device_request callback This flag is used in idigi_data_service_device_request callback telling server that callback did not process the message
#define IDIGI_MSG_RESP_SUCCESS   0x0100

Message success bit flag This flag is used in idigi_data_service_block_t indicating message successfully handled.

See also:
idigi_data_service_put_request callback This flag is used in idigi_data_service_put_request callback telling the callback that server successfully received the message.
#define IDIGI_MSG_BAD_REQUEST   0x0200

This flag is used to indicate that the message was invalid.

See also:
idigi_data_service_put_request callback This flag is used in idigi_data_service_put_request callback from server telling the callback that some portion of the data was invalid.
#define IDIGI_MSG_UNAVAILABLE   0x0400

This flag is used to indicate that the service is unavailable to process the message.

See also:
idigi_data_service_put_request callback This flag is used in idigi_data_service_put_request callback from server telling the callback that service is unavailable due to overload or other issues. Callback may try to resend the message later.
#define IDIGI_MSG_SERVER_ERROR   0x0800

This flag is used to indicate that the server encountered an error handling the message.

See also:
idigi_data_service_put_request callback This flag is used in idigi_data_service_put_request callback from server telling the callback that server encountered an error handling the message.
